## Эволюция взаимодействия с LLM

| Уровень | Описание | Паттерн |
|---------|----------|---------|
| Prompting | Вывод LLM не влияет на ход программы | `process_llm_output(llm_response)` |
| Router | LLM определяет базовый control flow | `if llm_decision(): path_a() else: path_b()` |
| Tool Calling | LLM выбирает функцию и аргументы | `run_function(llm_chosen_tool, llm_chosen_args)` |
| Multi-step Agent | Вывод LLM контролирует итерации | `while llm_should_continue(): execute_next_step()` |
| Multi-Agent System | Один агент может запускать другого | `if llm_trigger(): execute_agent()` |

## Что такое AI-агент

> **OpenAI Agents SDK:** агент — это AI-приложение, состоящее из модели с инструкциями, доступом к инструментам и рантаймом с динамическим жизненным циклом.

> **Lilian Weng:** Agent = LLM + memory + planning skills + tool use

Расширенная формула:

```
Agent = runtime(AI model + prompts + tools + memory + guardrails + planning skills)
```

Компоненты агента:
- **AI model** (LLM) — ядро, генерация текста
- **Prompts** — system, user, assistant
- **Tools** — функции для взаимодействия с внешним миром (Search, Calendar и др.)
- **Memory** — контекст, краткосрочная и долгосрочная память
- **Planning** — декомпозиция подзадач, рассуждение, рефлексия
- **Guardrails** — ограничения и фильтры безопасности
- **Agent runtime** — среда исполнения

## Что такое LLM

**Large Language Model (LLM)** — нейросетевая модель, обученная на больших текстовых корпусах, которая предсказывает следующий токен на основе контекста.

---

## Веса и предсказание следующего токена

### Токенизация

Три подхода к кодированию текста:
- **Символьное** — каждый символ отдельно (большие последовательности)
- **Словесное** — каждое слово отдельно (огромный словарь)
- **Субсловное (BPE и др.)** — баланс между длиной и словарём: `I`, `was`, `read`, `ing`, `interest`, `book`, `New_York`

### Специальные токены

Зарезервированные токены с предопределёнными функциями: `<eos>`, `<|endoftext|>`, `<|user|>`, `<|assistant|>`, `<|system|>`, `<translate>`, `<summarize>` и т.д.

### Авторегрессия

- Модель генерирует по одному токену за шаг
- Каждый новый токен добавляется в контекст для следующего шага
- Ограничение — длина контекстного окна

### Стратегии выбора токенов

- **Greedy** — наиболее вероятный токен
- **Top-k sampling** — выбор из k лучших кандидатов
- **Beam search** — исследование нескольких продолжений
- **Temperature** — контроль случайности (низкая: детерминированно; высокая: разнообразно)

## Как обучают LLM

| Этап | Метод | Данные | Результат |
| ---- | ----- | ------ | --------- |
| Pre-training | Self-supervised | Wiki, книги, веб-сайты | Foundational LLM |
| Instruction Fine-tuning (SFT) | Supervised | Инструкции и ответы от людей | SFT LLM |
| RLHF | Reinforcement Learning from Human Feedback | Оценки от аннотаторов | RLHF LLM |

## Сильные стороны и ограничения LLM

### LLM хороши в:
- Языковые задачи (суммаризация, перефразирование, генерация текста/кода)
- Структурированные паттерны рассуждений, использование инструментов, обработка файлов/таблиц

### LLM ограничены в:
- Точные алгоритмические/символьные задачи
- Актуальные знания (события после обучения)
- Длинное многошаговое планирование и персистентная память

> Многие ограничения можно компенсировать в агентах с помощью инструментов и памяти.

---

## Ключевые тезисы

- LLM оперируют **токенами**, а не словами или смыслами — это влияет на биллинг, лимиты контекста и структуру промптов
- LLM **предсказывают следующий токен** на основе вероятностей — отсюда температура и галлюцинации
- **Агент** = LLM + инструменты + память + планирование + guardrails + рантайм
